🧠 Solar2D ONNX Runtime HTML5 Plugin

This example demonstrates using ONNX Runtime Web with Solar2D's HTML5 JS bridge.

Note: This HTML file is for documentation. The actual plugin files are:
  • plugin_onnxruntime.lua — Lua wrapper
  • plugin_onnxruntime_js.js — JavaScript implementation

📋 Usage in Solar2D

local ort = require("plugin.onnxruntime")

-- Check if available (returns false on simulator/native)
if not ort.isAvailable() then
    print("ONNX Runtime not available")
    return
end

-- Load model (async with callback)
ort.load("model.onnx", function(session, error)
    if error then
        print("Load failed:", error)
        return
    end
    
    print("Model loaded!")
    local info = session:info()
    print("Inputs:", #info.inputs)
    print("Outputs:", #info.outputs)
    
    -- Prepare input tensor
    local inputs = {
        input = {
            dims = {1, 3, 224, 224},  -- NCHW format
            data = {0.5, 0.3, ...}     -- 150528 numbers
        }
    }
    
    -- Run inference (async with callback)
    session:run(inputs, function(outputs, error)
        if error then
            print("Inference failed:", error)
            session:close()
            return
        end
        
        -- Process results
        for name, tensor in pairs(outputs) do
            print("Output:", name)
            print("  Shape:", table.concat(tensor.dims, "x"))
            print("  First value:", tensor.data[1])
        end
        
        session:close()
    end)
end)

⚙️ Setup Instructions

  1. Copy the plugin files to your project's web/ directory
  2. Add to your build.settings: 
    plugins = {
    ["plugin.onnxruntime"] = {
        publisherId = "your.publisher.id",
    }
}
  3. Ensure your HTML5 template includes onnxruntime-web: 
    <script src="https://cdn.jsdelivr.net/npm/onnxruntime-web/dist/ort.min.js"></script>

🔧 API Differences from Native Plugin

Important: The HTML5 plugin uses callbacks due to JavaScript's async nature.
Operation Native (C) HTML5 (JS)
load local s = ort.load(path) ort.load(path, function(s, err) ... end)
run local o = s:run(inputs) s:run(inputs, function(o, err) ... end)
close s:close() s:close() (same)

🚀 Execution Providers

Pass execution provider in options:

-- WebGPU (fastest, if available)
ort.load("model.onnx", function(session) ... end, {
    ep = "webgpu"
})

-- WebGL (fallback)
ort.load("model.onnx", function(session) ... end, {
    ep = "webgl"
})

-- WebAssembly CPU (default, always works)
ort.load("model.onnx", function(session) ... end, {
    ep = "wasm",
    intraOpNumThreads = 4
})
